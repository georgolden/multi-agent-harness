### Design

Agent is not a universal session with skill or additional instructions

Agent is standalone session!

It has input output and conversation history

So for each agent you need to manage history and system prompt and tools

You need to display agent as it is


Right visualisation - board of agents that are available for any session discoverability and full control

Notifications of agents work

Chat with templates to adjust all the parameters


Runtime - dynamically add agents

Add agent to node runtime

Or just add agent via observability - file read. None node runtime


Code abstractions that are unavoidalbe but just write once

Tools - borrow from open source for CLI tools

Message history log for agent

Tool logger format to log tools
